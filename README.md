# STM32 Bare-Metal Human-Machine Interface (HMI)

A high-efficiency, zero-abstraction serial ecosystem designed for the **STM32L476RG** (ARM Cortex-M4) microcontroller. This project demonstrates advanced low-level embedded software engineering by implementing custom hardware peripheral drivers directly through register manipulation using **CMSIS** definitions, bypassing vendor-specific HAL/LL libraries.

The system features an interactive, hierarchical serial menu interface executed over UART, designed for predictable operator navigation via numerical inputs. This structure will serve as the core configuration terminal alongside a local Electronic Paper Display (ePaper) powered by SPI and a touchscreen interface managed over I2C.

---

## Technical Architecture

The firmware implements a layered architecture to decouple peripheral hardware control from the application's state machine logic.

```text
                  +---------------------------------+
                  |       Application Layer         |
                  |  (main.c - Hierarchical Menu)   |
                  +----------------+----------------+
                                   |
         +-------------------------+-------------------------+
         |                         |                         |
         v                         v                         v
+-----------------+       +-----------------+       +-----------------+
|   UART Driver   |       |   I2C Driver    |       |   SPI Driver    |
| (Serial Console)|       | (Touch/Sensors) |       | (ePaper Display)|
+-----------------+       +-----------------+       +-----------------+
         |                         |                         |
         +-------------------------+-------------------------+
                                   v
                  +---------------------------------+
                  |   CMSIS Register Definitions    |
                  +----------------+----------------+
                                   v
                  +---------------------------------+
                  |      STM32L476RG Hardware       |
                  +----------------+----------------+
```

### Core Architectural Choices
- **Menu Execution Model:** Finite State Machine (FSM) driven by predictable, single-byte numeric choices (`'1'`, `'2'`, `'3'`), optimizing processing overhead and eliminating complex string parsing.
- **Hardware Abstraction Level:** Bare-metal development utilizing solely CMSIS core headers for hardware register-map addressing.
- **Build System:** IDE-agnostic compilation via the GNU Arm Embedded Toolchain (`gcc-arm-none-eabi`) and **CMake**.

## Project Structure

The repository organizes source files into functional directories to maintain a strict separation of concerns, allowing the peripheral driver library to scale without impacting the application core:

```text
├── app/              # Main application state machines and HMI logic
├── drivers/          # Custom bare-metal peripheral drivers (UART, SPI, I2C, etc.) and CMSIS headers
├── include/          # Public driver interfaces and global configuration headers
├── scripts/          # Target-specific linker scripts and assembly startup files
└── CMakeLists.txt    # Toolchain and cross-compilation build automation
```

## Peripheral & Interface Implementation Status

### 1. UART Console & Menu System (Current Focus)
The UART module establishes the primary communication pipeline. The terminal renders pre-formatted text blocks representing menus and evaluates incoming bytes to trigger state transitions.
- **Configuration:** 115200 Baud, 8 Data Bits, 1 Stop Bit, No Parity (8N1).
- **Execution Model:** Direct register manipulation of the Universal Synchronous Asynchronous Receiver Transmitter (`USARTx`).
- **Navigation Logic:** 1. System transmits current menu options via UART.
  2. System awaits a numeric character input.
  3. The FSM evaluates the selection, routes to the corresponding submenu or execution routine, and re-renders the terminal interface.

*Future phases will incorporate SPI for ePaper graphics and I2C for touchscreen/sensor interfaces.*

## Development Environment & Workflow

- **Host OS:** Linux
- **IDE:** JetBrains CLion (Native CMake integration)
- **Deployment Strategy:** Compiled binaries (`.bin`/`.hex`) generated by the cross-compiler toolchain are flashed to the target **NUCLEO-L476RG** development board using STM32CubeProgrammer.
